# ggrpo

A high-performance, from-scratch implementation of **Group Relative Policy Optimization (GRPO)** in PyTorch, featuring custom fused Triton kernels for memory-efficient training. 

This repository implements the reinforcement learning algorithm used in DeepSeekMath to fine-tune causal language models (such as `Qwen2.5-1.5B-Instruct`) on code generation tasks.

---

## Features

- **Fused Triton Kernels**: Implements a custom, fused `get_per_token_logps` forward and backward pass using Triton 3.6.0. 
  - **Memory Efficiency**: Employs **activation recomputation** (similar to the tiling principles in FlashAttention) to avoid materializing the massive `[batch, seq_len, vocab_size]` intermediate logits or log-probability tensors in HBM (VRAM), saving gigabytes of memory during rollouts.
  - **Custom Autograd**: Fully integrated with PyTorch's autograd engine via a custom `torch.autograd.Function`, passing rigorous `torch.autograd.gradcheck` finite-difference validations.
- **Sandboxed Reward Execution**: Code generated by the model is executed in an isolated subprocess to evaluate correctness against unit tests safely.
- **RL Training Loop**: Implements REINFORCE-style group advantages, clipping, and KL divergence constraints to stabilize training.

## Installation & Setup

This project uses `uv` for lightning-fast python package and environment management.

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Ett05/ggrpo.git
   cd ggrpo
   ```

2. **Initialize environment and sync dependencies**:
   ```bash
   uv sync
   ```

3. **Install the package locally in editable mode**:
   ```bash
   uv pip install -e .
   ```

## Quickstart

### 1. High-Level GRPO Fine-tuning (`ggrpo.train`)

Fine-tune any causal LLM using GRPO in a single function call, powered by Triton memory kernels under the hood:

```python
import ggrpo
from transformers import AutoTokenizer, AutoModelForCausalLM
from ggrpo.dataset import problems
from ggrpo.reward import reward_function

model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-1.5B", torch_dtype="bfloat16").to("cuda")
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-1.5B")

# Fine-tune using ggrpo
history = ggrpo.train(
    model=model,
    tokenizer=tokenizer,
    dataset=problems,
    reward_fn=reward_function,
    group_size=4,
    num_epochs=10,
    lr=1e-5
)

# Plot rewards and loss curves
history.plot(save_path="rewards.png")
```

### 2. Low-Level Fused Logps Kernel (`ggrpo.get_per_token_logps`)

Use the low-level fused Triton kernel directly in your custom PyTorch training loops:

```python
import torch
import ggrpo

logits = torch.randn(4, 1024, 32002, device="cuda", requires_grad=True)
input_ids = torch.randint(0, 32002, (4, 1024), device="cuda")

# Fused Triton forward & backward pass (zero VRAM logits materialization)
logps = ggrpo.get_per_token_logps(logits, input_ids)
loss = logps.sum()
loss.backward()
```

## Testing & Benchmarking

To run the verification suite and verify the correctness/speedup of the custom Triton kernels:

```bash
uv run python ggrpo/kernels/test_kernel.py
```

This runs:
1. **Mathematical Validation**: Runs `torch.autograd.gradcheck` to compare the analytical gradients of the Triton backward pass against numerical gradients.
2. **Correctness Benchmarking**: Asserts that both Triton outputs and gradients match PyTorch eager values (tolerance $10^{-5}$).
3. **Execution Speed**: Compares and prints the speedup factor of the Triton kernel over PyTorch.

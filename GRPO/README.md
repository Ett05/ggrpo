# GRPO from Scratch

Implementing Group Relative Policy Optimization (GRPO) from scratch in PyTorch to fine-tune Qwen2.5-1.5B on code generation tasks.

## What this is

GRPO is a technique used in RL enviornments to train models. Essentially have the model genereate more than 1 answer and then evalue those answers using a reward function. Use the amalgamation to reward sequences that do better than average and conversly penalize sequences that do worse that average.


### Training loop
Generate N outputs for each problem in each epoch. Calculate the reward for each of those functions, additionally get the log probablity of each token. Rewards are normalized by subtracting the group mean and dividing by standard deviation before computing loss. Use the negative log probablity * the advantage to caclcuate loss of each token. Use that loss to backprop. 

### Reward function
In this implmentation it is a deterministic reward function which runs the code generated by the model in a seperate subprocess. If code runs correct (passes the test) then the sequence is reward (given reward score 1), if the code is unsuccesful the reward function gives reward 0 for that sequence. 

### Loss computation
Add KL and clipping logic to ensure at each token the model isn't diverging. KL compares token probablites with the model in the beginning whereas clipping ensures that each iteration the model isn't diverging too much.
